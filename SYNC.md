# Drawlatch-Callboard Secure Key Exchange (Sync Protocol)

Design document for the double-code mutual approval key exchange between a
drawlatch remote server and a callboard instance.

---

## Overview

When a callboard instance wants to connect to a drawlatch remote server, both
sides need each other's public keys before the normal encrypted handshake can
occur. The sync protocol bootstraps this trust relationship using a
human-mediated double-code flow with an additional encryption key to protect
the exchange in transit.

**Goal**: After a successful sync, both sides have each other's public keys
registered and the caller is configured on the remote server — ready for
normal encrypted MCP communication.

---

## Protocol Flow

```
DRAWLATCH (remote server)              USER              CALLBOARD (imports drawlatch)
--------------------------              ----              ----------------------------
1. drawlatch sync
   -> invite code:    WOLF-3847
   -> encryption key: (base64 AES key)
   -> starts /sync listener             --copy both-->
                                                        2. startKeyExchange(inviteCode, encKey, endpoint)
                                                           -> generates caller keys (if needed)
                                                           -> confirm code: BEAR-9021
                                                           -> displays confirm code
                                          <--copy--
3. User enters confirm code
   -> server now expects a call
      bearing both codes
                                                        4. POST /sync
                                                           Body: AES-encrypted JSON:
                                                           { inviteCode, confirmCode,
                                                             callerAlias, publicKeys }
5. Decrypt body with encryption key
   Validate: inviteCode matches
   Validate: confirmCode matches
   -> save callboard's public keys as peer
   -> register caller in config
   -> respond with AES-encrypted JSON:
      { remotePublicKeys, callerAlias,
        configuredConnections }
                                                        6. Decrypt response
                                                           -> save remote public keys
                                                           -> sync complete
7. Sync listener closes
```

### Security Properties

- **Double-code verification**: Both sides generate a code; the user transfers
  each out-of-band. An attacker would need to intercept both codes.
- **Encryption key**: A one-time AES-256-GCM key generated by drawlatch and
  transferred out-of-band with the invite code. The sync payload is encrypted
  end-to-end — even if the network is untrusted.
- **Single-use**: The sync listener accepts exactly one successful exchange,
  then shuts down. Codes cannot be replayed.
- **Time-limited**: Sync sessions expire after 5 minutes (configurable).
- **No sync without listening**: The `/sync` endpoint returns 404 unless a
  sync session is actively open. There is no persistent sync surface.
- **Public keys only**: Private keys never leave their host.

## Components

### 1. Key Manager API (`src/shared/crypto/key-manager.ts`)

High-level CRUD over the drawlatch key directory structure. Callboard imports
this to manage keys programmatically without touching the filesystem directly.

```typescript
import { ... } from '@wolpertingerlabs/drawlatch/shared/crypto';

// Create a new caller identity (generates Ed25519 + X25519 keypairs)
createCaller(alias: string, opts?: { configDir?: string }):
  { publicKeys: SerializedPublicKeys, fingerprint: string }

// Export public keys for a local identity or the remote server
exportPublicKeys(type: 'local' | 'remote', alias?: string, opts?: { configDir?: string }):
  SerializedPublicKeys

// Import a peer's public keys (saves under keys/peers/<alias>/)
importPeerPublicKeys(alias: string, keys: SerializedPublicKeys, opts?: { configDir?: string }):
  void

// Export a peer's previously-imported public keys
exportPeerPublicKeys(alias: string, opts?: { configDir?: string }):
  SerializedPublicKeys

// Save public keys from a remote server (saves under keys/peers/remote-server/)
saveRemotePublicKeys(keys: SerializedPublicKeys, opts?: { configDir?: string }):
  void

// List all local caller aliases
listCallers(opts?: { configDir?: string }): string[]

// List all imported peer aliases
listPeers(opts?: { configDir?: string }): string[]

// Check if a caller identity exists
callerExists(alias: string, opts?: { configDir?: string }): boolean

// Check if a peer's keys are imported
peerExists(alias: string, opts?: { configDir?: string }): boolean
```

**Directory structure** (under `~/.drawlatch/keys/`):
```
keys/
  local/<alias>/        # Full keypairs (pub + private) for local identities
  remote/               # Full keypairs for the remote server identity
  peers/<alias>/        # Public keys only for imported peers
    signing.pub.pem
    exchange.pub.pem
```

### 2. Sync Protocol (`src/shared/protocol/sync.ts`)

Shared types, code generation, and encryption helpers used by both sides.

```typescript
// --- Code generation ---

// Human-readable sync code: WORD-NNNN (e.g., "WOLF-3847")
generateSyncCode(): string

// One-time AES-256-GCM key for encrypting the sync exchange
generateSyncEncryptionKey(): string   // base64-encoded 32 bytes

// --- Encryption ---

// Encrypt a JSON payload with the sync encryption key
encryptSyncPayload(payload: object, encryptionKey: string): string  // base64

// Decrypt a sync payload back to JSON
decryptSyncPayload(encrypted: string, encryptionKey: string): object

// --- Types ---

interface SyncRequest {
  inviteCode: string;           // Generated by drawlatch
  confirmCode: string;          // Generated by callboard
  callerAlias: string;          // Requested caller alias
  publicKeys: SerializedPublicKeys;  // Callboard's public keys
}

interface SyncResponse {
  remotePublicKeys: SerializedPublicKeys;  // Drawlatch's public keys
  callerAlias: string;                     // Confirmed caller alias
  fingerprint: string;                     // Remote key fingerprint
}

interface SyncSession {
  inviteCode: string;
  confirmCode: string | null;   // Set when user enters it
  encryptionKey: string;
  createdAt: number;
  ttlMs: number;
  completed: boolean;
}

type SyncErrorCode =
  | 'NO_ACTIVE_SESSION'         // /sync called but nobody is listening
  | 'CODE_MISMATCH'             // invite or confirm code doesn't match
  | 'SESSION_EXPIRED'           // TTL exceeded
  | 'ALREADY_COMPLETED'         // sync already used
  | 'DECRYPTION_FAILED'         // couldn't decrypt with the encryption key
  | 'INVALID_PAYLOAD';          // decrypted but malformed
```

### 3. Sync Client API (`src/shared/protocol/sync-client.ts`)

Programmatic API for callboard to drive its side of the exchange.

```typescript
import { ... } from '@wolpertingerlabs/drawlatch/shared/protocol';

interface KeyExchangeInit {
  confirmCode: string;                    // Display this to the user
  complete(): Promise<SyncResult>;        // Call after user enters confirm code into drawlatch
}

interface SyncResult {
  remotePublicKeys: SerializedPublicKeys;
  callerAlias: string;
  fingerprint: string;
}

// Start the key exchange from callboard's side
//
// 1. Creates caller keys if they don't exist
// 2. Generates a confirm code
// 3. Returns the confirm code + a complete() function
// 4. complete() encrypts and sends POST /sync, decrypts response, saves keys
//
startKeyExchange(opts: {
  remoteUrl: string;            // Drawlatch endpoint (e.g., "http://host:9999")
  inviteCode: string;           // From drawlatch sync output
  encryptionKey: string;        // From drawlatch sync output
  callerAlias: string;          // Desired caller alias
  configDir?: string;           // Override config dir (default: ~/.drawlatch)
  connections?: string[];       // Requested connection aliases (informational)
}): Promise<KeyExchangeInit>
```

**Usage in callboard** (pseudocode):
```typescript
// User pastes invite code + encryption key from drawlatch
const exchange = await startKeyExchange({
  remoteUrl: 'http://192.168.1.10:9999',
  inviteCode: 'WOLF-3847',
  encryptionKey: 'base64...==',
  callerAlias: 'my-callboard',
});

// Display to user: "Enter this code into drawlatch: BEAR-9021"
console.log(`Confirm code: ${exchange.confirmCode}`);

// User enters BEAR-9021 into drawlatch, then callboard calls complete()
const result = await exchange.complete();
// -> Keys exchanged, remote public keys saved locally
```

### 4. Remote Server Sync Endpoints (`src/remote/server.ts`)

Two internal endpoints added to the Express server:

#### `POST /sync/listen` (internal management)

Called by the drawlatch CLI to open a sync session on the running server.

```
Request:  { inviteCode, confirmCode, encryptionKey, ttlMs? }
Response: { ok: true }
```

- Stores the session in memory (not persisted)
- Only one sync session at a time
- Session auto-expires after `ttlMs` (default: 300000 = 5 min)

#### `POST /sync` (external — called by callboard)

```
Request body:  base64-encoded AES-encrypted SyncRequest
Response body: base64-encoded AES-encrypted SyncResponse
```

**Behavior**:
1. If no sync session is active: **404** `{ error: 'NO_ACTIVE_SESSION' }`
2. Decrypt request body with the session's encryption key
   - Failure: **400** `{ error: 'DECRYPTION_FAILED' }`
3. Validate `inviteCode` matches session
   - Failure: **403** `{ error: 'CODE_MISMATCH' }`
4. Validate `confirmCode` matches session
   - Failure: **403** `{ error: 'CODE_MISMATCH' }`
5. Check session not expired
   - Failure: **410** `{ error: 'SESSION_EXPIRED' }`
6. Check session not already completed
   - Failure: **409** `{ error: 'ALREADY_COMPLETED' }`
7. Save callboard's public keys as a peer under `keys/peers/<callerAlias>/`
8. Register the caller in `remote.config.json` (with empty connections list —
   admin configures connections separately)
9. Encrypt and return `SyncResponse` with remote server's public keys
10. Mark session as completed; close sync listener

### 5. Drawlatch CLI `sync` Command (`bin/drawlatch.js`)

New subcommand: `drawlatch sync`

**Interactive flow**:
```
$ drawlatch sync

Starting key exchange...

  Invite code:    WOLF-3847
  Encryption key: dGhpcyBpcyBhIHRlc3Qga2V5IGZvciBkZW1v...

Give both values to the callboard operator.
They will provide a confirm code.

Enter confirm code: BEAR-9021

Waiting for sync from callboard...

Sync complete!
  Caller alias:  my-callboard
  Fingerprint:   a3:f2:1b:9e:44:...
  Keys saved to: ~/.drawlatch/keys/peers/my-callboard/

Add connections for this caller in remote.config.json:
  "callers": {
    "my-callboard": {
      "connections": ["github", "slack", ...]
    }
  }
```

**Options**:
```
drawlatch sync [options]

Options:
  --ttl <seconds>    Sync session timeout (default: 300)
  -h, --help         Show help
```

**Implementation**: The CLI:
1. Generates invite code + encryption key via `generateSyncCode()` / `generateSyncEncryptionKey()`
2. Displays them
3. Prompts for confirm code (readline)
4. Sends `POST /sync/listen` to the running server with all three values
5. Polls `GET /sync/status` until completed or expired
6. Displays result

If the server is not running, the CLI prints an error telling the user to
start it first (`drawlatch start`).

#### `GET /sync/status` (internal management)

Returns the current sync session state for CLI polling:

```
Response: { active: boolean, completed: boolean, callerAlias?: string, fingerprint?: string }
```

## Security Considerations

1. **No persistent attack surface**: `/sync` returns 404 when no session is
   active. The endpoint only exists during the brief window when an admin
   has explicitly run `drawlatch sync`.

2. **Three out-of-band secrets**: The invite code, encryption key, and confirm
   code are all transferred via a side channel (copy-paste, secure message,
   etc.). An attacker would need to compromise the side channel.

3. **Encryption in transit**: Even if the network between callboard and
   drawlatch is untrusted, the sync payload is AES-256-GCM encrypted with
   the out-of-band key. The attacker cannot read or tamper with the exchange.

4. **Single-use + time-limited**: Each sync session accepts exactly one
   successful exchange and expires after 5 minutes. No replay possible.

5. **Minimal trust granted**: The sync only registers the caller with an empty
   connections list. The admin must explicitly grant access to specific
   connections after the sync completes.

6. **Public keys only**: Private keys never leave their respective hosts.
   Only public signing and exchange keys are transmitted.

## Future Considerations

- **Batch sync**: Allow multiple callboard instances to sync in sequence
  without restarting drawlatch sync each time.
- **QR code**: Encode invite code + encryption key as a QR code for easier
  transfer.
- **Revocation**: `drawlatch revoke <alias>` to remove a caller's keys and
  config entry.
- **Re-sync**: Handle the case where a caller already exists but needs
  updated keys (key rotation).
